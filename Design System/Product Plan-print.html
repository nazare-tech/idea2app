Product Brief

Introduction & Overview

01 / 09

Instead of imagining how a garment might look on their specific body, users see themselves in it instantly, on any site they browse.

The product solves a critical pain point for fashion influencers: style confidence. Influencers must curate compelling content and affiliate recommendations without purchasing every item they consider. The timing is right because generative image-to-image diffusion models have recently crossed the photorealism threshold that influencers require to trust and share outputs publicly.

The MVP targets fashion influencers on desktop web via a Chrome-compatible browser extension, monetized through pay-per-use credits.

Today's tools don't follow the user across the open web

Veesual
Locked to specific retailer partnerships.
Google VTO
Restricted to pre-set models.
ZMO.ai
Manual uploads, no browsing integration.

Wear It Now AI Mirror is the first consumer-controlled, universal try-on layer that works on any clothing website.

User goals

  • See themselves wearing any garment on any clothing website, without buying or uploading each session.
  • Quickly evaluate whether an item suits their style and body before committing.
  • Generate high-quality, shareable try-on images for content and engagement.
  • Browse multiple retailers in one session with a consistent, personalized experience.

Non-goals

  • Mobile browsers or native mobile apps.
  • Non-clothing categories (shoes, accessories, bags).
  • A standalone web app for manual image uploads.
  • B2B retailer integration or white-label product.
  • Video or animated try-on.
  • Social feed, community, or follower system.
  • AI styling recommendations or advice.
  • Multiple saved avatar profiles per user.

Technical Considerations

06 / 09

Existing systems

  • Greenfield product. No existing codebase to integrate with.
  • The extension talks to a backend API hosted separately from the marketing site.

Authentication

  • Email/password and Google OAuth via a managed auth provider (assumed Supabase or Firebase Auth).
  • The extension stores a JWT or session token in chrome.storage.local for persistent login.

AI generation API

  • Core try-on depends on an image-to-image model for realistic garment transfer (assumed Replicate, Fashn.ai, or a custom diffusion endpoint).
  • API accepts a product image URL and avatar reference, returns a generated image URL or blob.
  • Assumed latency 3–10s per image, communicated via loading states.

Data storage

  • Database: relational (assumed PostgreSQL via Supabase) for users, balances, transactions, history, and reports.
  • Entities: users, avatar_photos, credit_transactions, generation_sessions, generated_images, site_reports.
  • Files: avatars and generated images in object storage (assumed 30-day retention for caching).

Payments

  • Stripe for credit bundles. Webhooks update balances server-side on confirmed payment to prevent client-side manipulation.

Extension architecture

  • Manifest V3 Chrome extension.
  • Content script injected into retailer pages to scan the DOM and replace images.
  • Background service worker for API calls and session state; popup in a lightweight framework (React/Preact).

Privacy & security

  • Avatar photos are biometric-adjacent. The privacy policy must disclose storage, processing, and retention.
  • No model training on avatars without explicit consent. Generated images are scoped to the owning account.
  • All API traffic over HTTPS. Minimum Chrome permissions: activeTab, storage, identity.

Performance

  • Popup opens within 500ms; DOM scan under 2s for up to 50 images.
  • Each generation under 10s; over 30s triggers timeout with retry.
  • Marketing site & dashboard target Lighthouse 80+. Balance updates within 3s of a confirmed action.

Analytics

  • Assumed PostHog, configured to track the events defined in the functional requirements.

Scalability

  • MVP scale: up to 5,000 registered users and 500 concurrent generation requests. Evaluate AI provider rate limits at this scale.
