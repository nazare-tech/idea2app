Executive Summary

Market Snapshot & Entry Thesis

The AI Virtual Try-On (VTO) category is rapidly evolving, primarily driven by B2B solutions that empower retailers to showcase clothing on diverse AI models. However, a significant market gap exists for consumer-controlled, cross-platform VTO tools that operate independently of retailer integrations. By utilizing a browser extension model, Wear It Now AI Mirror shifts the power dynamic directly to fashion influencers, offering a highly personalized, universal try-on experience that bypasses traditional B2B bottlenecks.

Opportunity Verdict

Opportunity Verdict

High-potential consumer wedge in B2B-dominated market

  1. 01

    Verdict: Strong opportunity to capture the influencer niche by decentralizing virtual try-on technology through a universal browser extension.

  2. 02

    Why Now: Generative AI image-to-image translation (like diffusion models) has recently crossed the threshold of photorealism required for fashion influencers to trust and share the outputs.

  3. 03

    Biggest Risk: Clothing retailers frequently update their site structures or employ anti-scraping technologies, which could break the browser extension's ability to parse and replace product images seamlessly.

  3. 03

Risks & Competitor Responses

  1. 01

    Retailer Retaliation: Brands may view the extension as copyright infringement or unauthorized modification of their intellectual property and implement DOM obfuscation to break the extension.

  2. 02

    Big Tech Native Integration: Browsers (like Edge or Chrome) or OS-level AI (Apple Intelligence) could introduce native virtual try-on, instantly commoditizing third-party extensions.

  3. 03

    Quality Churn: If the AI fails to accurately represent fabric physics (e.g., making a stiff jacket look like a soft shirt), fashion influencers will immediately abandon the tool to protect their credibility.

  4. 04

    Competitor Pivot: B2B players like ZMO.ai could easily release a consumer-facing browser extension using their existing, highly-trained enterprise rendering models.

Product Brief

Introduction / Overview

Wear It Now AI Mirror is a desktop browser extension powered by generative AI that transforms any clothing retailer's website so that every product image is replaced in real time with a photo-realistic rendering of the user wearing those clothes. Instead of imagining how a garment might look on their specific body, users see themselves in it instantly, on any site they browse.

The product solves a critical pain point for fashion influencers: style confidence. Influencers must curate compelling content and affiliate recommendations without purchasing every item they consider. Today's virtual try-on tools are either locked to specific retailer partnerships (Veesual), restricted to pre-set models (Google VTO), or require manual image uploads with no browsing integration (ZMO.ai). None of them follow the user across the open web. Wear It Now AI Mirror is the first consumer-controlled, universal try-on layer that works on any clothing website.

The timing is right because generative AI image-to-image diffusion models have recently crossed the photorealism threshold that influencers require to trust and share outputs publicly. The MVP targets fashion influencers on desktop web via a Chrome-compatible browser extension, monetized through pay-per-use credits.

Non-goals

Role-Based Access

Guest (unauthenticated): Can install the extension and view the product landing page. Cannot activate the try-on overlay. Cannot generate any try-on images. Is prompted to register when attempting to activate the extension.

Registered user (free, no credits): Can log in, upload and save one avatar photo, and view their credit balance. Can activate the extension on any clothing site but will be prompted to purchase credits before any image is generated. Can access account settings and purchase credits.

Registered user (with credits): Full access to the try-on overlay on any supported clothing website. Each image generation consumes one credit. Can download generated images. Can view generation history.

Admin: Can view all registered users, credit balances, and transaction history. Can manually adjust credit balances. Can flag or review generated images for policy violations. Can manage site compatibility reports submitted by users.

As a fashion influencer, I want to install the Wear It Now AI Mirror browser extension and activate it on a clothing website, So that I can see myself wearing the products on that site.

As a new visitor, I want to create an account using my email or Google, So that I can access the try-on features.

  • The user is redirected to the onboarding avatar upload flow after registration.

Technical

Technical Considerations

Existing systems and integrations

  • The product is greenfield. No existing codebase to integrate with.
  • The browser extension communicates with a backend API hosted separately from the marketing site.

Authentication

  • Email/password authentication and Google OAuth via a managed auth provider (assumed: Supabase Auth or Firebase Auth) to reduce implementation overhead.
  • The extension stores a JWT or session token in Chrome's chrome.storage.local for persistent login.

AI generation API

  • The core try-on capability depends on an image-to-image generative AI model capable of realistic garment transfer. Assumed integration with a third-party API (e.g., Replicate, Fashn.ai, or a custom-hosted diffusion model endpoint) rather than training a proprietary model for MVP.
  • The API must accept a product image URL and a user avatar image reference and return a generated image URL or base64 blob.
  • Assumed generation latency: 3–10 seconds per image. This must be communicated clearly to users via loading states.

Data storage

  • Database: Relational database (assumed: PostgreSQL via Supabase) for users, credit balances, transactions, generation history, and site compatibility reports.
  • Key entities: users, avatar_photos, credit_transactions, generation_sessions, generated_images, site_reports.
  • File storage: Avatar photos and generated images stored in object storage (assumed: Supabase Storage or AWS S3). Generated images may be stored temporarily (assumed: 30-day retention) to support session caching and history.

Payments

  • Stripe for credit bundle purchases. Stripe webhooks must update credit balances server-side upon confirmed payment to prevent client-side manipulation.

Browser extension architecture

  • Manifest V3 Chrome extension.
  • Content script injected into retailer pages to scan DOM and replace images.
  • Background service worker to manage API calls and session state.
  • Extension popup built with a lightweight framework (assumed: React or Preact).

Privacy and security

  • Avatar photos are sensitive biometric-adjacent data. The privacy policy must clearly disclose how avatar photos are stored, processed, and retained.
  • Avatar photos must not be used to train AI models without explicit user consent.
  • Generated images must be stored under the user's account and must not be accessible to other users.
  • All API communication must use HTTPS.
  • The extension must request only the minimum required Chrome permissions: activeTab, storage, identity (for OAuth).

Performance expectations

  • Extension popup must open within 500ms of clicking the icon.
  • Page DOM scan must complete within 2 seconds on pages with up to 50 product images.
  • Individual try-on image generation must complete within 10 seconds under normal load. Images exceeding 30 seconds trigger a timeout with a retry option.
  • The marketing website and account dashboard must achieve a Lighthouse performance score of 80 or above.
  • Credit balance must update in the extension popup within 3 seconds of a confirmed generation action.

Analytics

  • Assumed: PostHog for product analytics, configured to track the events defined in functional requirements 41–46.

Scalability

  • MVP scale assumed: up to 5,000 registered users and up to 500 concurrent generation requests. The AI API provider must be evaluated for rate limits at this scale.

Scope

Non-goals / Out Of Scope

  • Mobile browser or native app support: The extension model requires a desktop browser. Mobile is deferred until the core experience is validated.
  • Firefox, Safari, or Edge support: The MVP targets Chrome only to reduce QA and compatibility overhead. Other browsers will be evaluated post-launch.
  • Non-clothing categories (shoes, accessories, bags, jewelry): Garment try-on is technically distinct from footwear and accessories. These require separate model training and are deferred.
  • Video or animated try-on: Real-time video overlay requires significantly more compute and latency management. Deferred to a future version.
  • Multiple avatar profiles per account: Users may want separate avatars for different body presentations. This adds complexity to the credit model and UX and is deferred.
  • Outfit saving, wishlists, or lookbooks: Content organization features are valuable but not required to validate the core try-on experience.
  • Social sharing, community feed, or follower system: Social features are a growth layer, not an MVP requirement.
  • AI styling recommendations: Suggesting what to wear based on preferences is a separate product capability deferred post-MVP.
  • B2B retailer integration or white-label product: The MVP is entirely consumer-facing. B2B is a future revenue stream.
  • Subscription pricing model: The MVP uses pay-per-use credits only. Subscriptions may be introduced once usage patterns are understood.
  • Affiliate link tracking or monetization tools: Influencer-specific affiliate features are deferred to a later phase.
  • Offline mode: The extension requires an active internet connection to call the AI generation API.

Thesis

MVP Summary

Wear It Now AI Mirror is a Chrome browser extension that replaces product images on clothing websites with AI-generated photos of the user wearing those clothes. It targets fashion influencers (10k–500k followers) who need to evaluate and showcase garments across multiple retailers without purchasing them. The core problem is style confidence: flat product photos don't show how clothes look on a specific body. This MVP will validate one critical assumption before anything else: can the AI output quality reach a threshold that influencers would trust enough to download, share, or use in content? If the output looks cheap or uncanny, the entire product collapses regardless of how well the extension is built.

Scope Decisions

Key Assumptions And Scope Decisions

  • [HIGH CONFIDENCE] Fashion influencers are the primary user. All scope decisions prioritize their workflow.
  • [VALIDATION DECISION] A full browser extension is not the right first step. AI output quality is the riskiest assumption. The first version is a standalone web page where users upload a photo of themselves and a product image, and the AI generates a try-on result. This tests output quality before investing in extension infrastructure.
  • [ASSUMPTION] A single avatar photo per user is sufficient for MVP. The PRD confirms no multiple saved profiles.
  • [SCOPE DECISION] The PRD lists a full role-based access system (guest, registered, admin), generation history, download tracking, and share analytics. The MVP will include only: avatar upload, single try-on generation, credit purchase, and result download. Everything else is deferred.
  • [SCOPE DECISION] The browser extension is the target delivery format but is deferred to Phase 2. The MVP is a web app that proves output quality and willingness to pay. Extension infrastructure adds 2–4 weeks of build time and Chrome Web Store review risk before the core assumption is validated.
  • [COMPLIANCE FLAG] Users upload photos of themselves. This is biometric/personal image data. Do not store raw uploaded photos beyond the session unless the user explicitly opts in. Generate and store only the output image. Add a clear data use disclosure at upload. Do not use uploaded photos to train models without explicit consent. Defer any persistent avatar storage until privacy policy and data handling are reviewed.
  • [CONFLICT RESOLVED] The PRD says "no standalone web app for manual image uploads" as a non-goal, but the intake recommends a browser extension and the riskiest assumption is AI output quality. Overriding the non-goal for MVP only: a minimal web-based prototype is the fastest way to test output quality without Chrome Web Store delays. The extension remains the target. This is a sequencing decision, not a product direction change.

Validation

MVP Goal, Definition Of Done, And Riskiest Assumptions

Goal: Validate whether AI-generated try-on output quality is high enough that fashion influencers would download, share, or use the result in content — and whether they will pay credits to generate more.

Definition of Done: A user can upload their photo, upload or paste a product image URL, generate a try-on result, and download it — without help — and the riskiest assumption has been testably exposed.

Riskiest Product Assumption: The AI output is realistic enough that influencers trust it for content creation or affiliate showcasing. If outputs look uncanny or low-quality, users will not pay for credits regardless of UX.

Riskiest Technical Assumption: The image-to-image generation model (e.g., via Replicate or a fine-tuned diffusion model) produces consistent, photorealistic garment-on-person results within an acceptable latency window (under 30 seconds) using only a single reference photo of the user and a product image.

Audience

Target User And Problem

Primary User: Mid-tier fashion influencers (10k–500k followers) who post outfit content on Instagram or TikTok and earn income through affiliate links.

Problem: They cannot evaluate how a garment looks on their specific body from flat product photos, which means they either buy items speculatively (wasted spend) or skip recommending items they're unsure about (lost affiliate revenue).

Current Workaround: Buy the item, photograph it, return it if it doesn't work. Or skip it entirely. Some use generic virtual try-on tools tied to specific retailers, which don't cover the brands they actually shop.

Desired Outcome: See a realistic, shareable image of themselves wearing any garment from any retailer, fast enough to use in their content workflow without purchasing first.
