# Maker Compass Operating System

This folder defines the working agreements for turning research, meetings, and team discussion into durable context and Linear work.

## Source Of Truth

- Raw transcripts are private inputs only. Do not commit them, copy them into Linear, or post them to Slack.
- `docs/research/` stores sanitized research summaries, processed insights, and research-driven decisions.
- `docs/meetings/` stores sanitized meeting notes and meeting-driven decisions.
- Linear is the source of truth for actionable work, owners, priority, status, due dates, and acceptance criteria.
- Slack is for discussion and notifications. Link to Linear or sanitized repo docs instead of storing tasks there.

## Required Protocols

- Read `transcript-sanitization-protocol.md` before handling any raw research or meeting transcript.
- Use `research-format.md` for research summaries.
- Use `meeting-notes-format.md` for meeting summaries.
- Use `linear-issue-format.md` when extracting action items into Linear.

## Intake Rule

When a raw transcript is provided, ask for missing metadata before processing:

- When did this research or meeting happen?
- For research: who was the participant?
- For meetings: what was the meeting title or topic, and who attended?

If the user already provided the metadata, proceed without asking again.
