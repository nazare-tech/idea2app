# Standardized Intake Test Cases

Use these fixtures whenever you need repeatable Maker Compass project-intake inputs, generated-report comparisons, prompt-lab context, UI checks, or manual QA for `/projects/new`.

Default rule: use **Idea 1.1** unless the user explicitly asks for another variant, or unless comparing multiple generated projects would materially help the task.

The intake wizard asks AI-generated follow-up questions, so the exact wording and option IDs can change. Prefer the answer intent below over exact text matching, and append newly observed question/answer pairs to the log at the bottom of this file.

## How To Use

1. Open `/projects/new`.
2. Paste the selected idea into Step 1.
3. When Step 2 questions appear, answer using the matching fixture below.
4. Always answer the required platform question using the canonical `primary-platform` option ID listed for the idea.
5. If the UI asks a new question, choose the closest answer from the idea's answer policy, then append the question and chosen answer to `Observed Question Log`.
6. For question-capture QA, stop before clicking `Create project` unless the task explicitly asks for full project creation.
7. For loading, progressive generation, onboarding progress, derived-readiness, or generated-document structure QA, click `Create project` and use that newly generated project as the primary evidence. Capture the relevant transient state while generation is active; do not substitute an older saved project.
8. Older projects may be used as supplemental backward-compatibility evidence only. Label their creation age and the legacy state they demonstrate so they are not mistaken for current-pipeline verification.

Do not use Ideas 1.2, 2.1, 2.2, 3.1, or 3.2 by default. They exist for explicit alternate-domain testing or compare/contrast runs.

## Idea 1: B2B Product Intelligence

Status: **default domain**. Use **Idea 1.1** as the default variant.

Use this idea for most tests.

```text
A B2B product intelligence platform that ingests support tickets, chat transcripts, sales calls, product analytics notes, and internal docs to surface recurring customer pain, detect trend shifts, and recommend roadmap priorities for product managers and customer-facing teams.
```

### Stable Answer Policy

| Question category | Default answer |
| --- | --- |
| Target audience | Product managers, product ops, and customer-success leaders at B2B SaaS companies. |
| Problem intensity | High urgency / painful recurring workflow. |
| Key workflow or use case | Centralize customer feedback, cluster themes, quantify impact, and turn findings into roadmap recommendations. |
| Business model | B2B SaaS subscription, priced by workspace or seat with usage limits for connected sources. |
| Launch priority | Start with integrations, insight clustering, source citations, and roadmap-priority scoring. |
| Differentiation | Explainable recommendations with evidence links back to source conversations and tickets. |
| Data sources | Support tickets, sales calls, chat transcripts, NPS notes, product docs, and CRM notes. |
| Buyer or decision maker | VP Product, Head of Product Ops, or CX/Product leadership. |
| First version scope | Connect 2-3 sources, generate weekly theme reports, and export prioritized opportunities. |
| Success metric | Time saved synthesizing feedback and number of roadmap decisions backed by customer evidence. |
| Primary platform | `primary-platform`: `desktop-web` / Desktop website. |

### Starter Follow-Up Answers

Use these answers when the generated questions match the common intake categories.

| Starter question | Answer to use |
| --- | --- |
| Who is the first target customer? | Product managers and product-ops teams at B2B SaaS companies with at least 500 monthly support or sales interactions. |
| What customer problem should the first version solve? | Teams cannot reliably see which customer pain points are recurring, revenue-relevant, and worth prioritizing. |
| Which workflow matters most for launch? | Import feedback from support and call transcripts, cluster themes, show evidence, and recommend roadmap priorities. |
| How will this business make money? | Monthly B2B SaaS subscription with tiers based on connected sources, seats, and analysis volume. |
| What should the first release optimize for? | Trustworthy synthesis: evidence-backed insights, clear scoring, and fast weekly reporting over broad collaboration features. |
| Where will people use the first version? | Desktop website (`desktop-web`). |

## Question Matching Guide

When the UI asks a question not listed above, map it to the closest category:

| If the UI asks about... | Use this fixture field |
| --- | --- |
| Customer, user, audience, persona, buyer | Target audience, buyer or decision maker |
| Pain, urgency, frequency, current alternative | Problem intensity and customer problem |
| Workflow, first use case, core action, job to be done | Key workflow or first version scope |
| Monetization, pricing, GTM, sales motion | Business model |
| MVP, launch focus, first release, must-have feature | Launch priority and first version scope |
| Competitive edge, moat, why now | Differentiation |
| Integrations, inputs, uploaded content, source material | Data sources |
| Platform, device, app type, where users open it | Primary platform canonical option ID |
| Metrics, success, validation | Success metric |

If the UI presents fixed chips, choose the closest chip by meaning. If there is an `Other` field and none of the chips match, choose `Other` and enter the relevant default answer in plain English.
